Syntax

A AgentMark file has two primary components to it.

  1. Frontmatter: Contains your prompt configuration
  2. Body: Contains messages, and notes about the prompt

Frontmatter​

The frontmatter contains most of the metadata about your prompt. At a high-level, this includes.

PropertyTypeDescriptionOptional/Required
namestringThe name of the prompt, serving as an identifier.Required
metadataMetadataSchemaMetadata associated with the prompt, containing model configuration details.Required

MetadataSchema Properties

PropertyTypeDescriptionOptional/Required
modelModelConfigSchemaConfiguration details of the language model, including its name and settings.Required

ModelConfigSchema Properties

PropertyTypeDescriptionOptional/Required
namestringThe name of the language model to be used.Required
settingsModelSettingsConfiguration settings for the model, which can be one of the following:Optional

This structured breakdown outlines the hierarchical properties of the AgentMarkSchema, detailing their types, descriptions, and whether they are optional or required.

Model Settings​

Model settings are standardized across models to allow for seamlessly switching between different providers.

Note: Not every feature is supported across models.

Common Settings​

PropertyTypeDescriptionOptional/Required
streambooleanIndicates whether to stream the response.Optional
max_tokensnumberMaximum number of tokens to generate.Optional
temperaturenumberControls the randomness of the output; higher values result in more random outputs.Optional
top_pnumberControls the cumulative probability for nucleus sampling.Optional
top_knumberLimits the next token selection to the top k tokens.Optional
presence_penaltynumberPenalizes new tokens based on their presence in the text so far, encouraging the model to discuss new topics.Optional
frequency_penaltynumberPenalizes new tokens based on their frequency in the text so far, reducing the likelihood of repeating the same line verbatim.Optional
stop_sequencesstring[]Array of strings where the generation will stop if any of the strings are encountered.Optional
seednumberSeed value for random number generation, ensuring reproducibility.Optional
max_retriesnumberMaximum number of retries for the request in case of failures.Optional
headersRecord<string, string>Additional headers to include in the request.Optional

Text Output Settings​

PropertyTypeDescriptionOptional/Required
toolsRecord<string, { description: string; parameters: JSONObject; }>A record of tools available to the model, where each tool includes: - description: A brief description of the tool's functionality. - parameters: Parameters required by the tool, defined as a JSON object.Optional

Object Output Settings​

PropertyTypeDescriptionOptional/Required
schemaJSONObjectA schema defining the expected structure of the output object.Required

Body​

The body of your prompt corresponds to the messages being sent to the LLM.

Message Tags​

Prompts support the use of 3 different message related tags:

  1. System
  2. User
  3. Assistant

These tags will be sent to the model provider in the order they are used. Any text placed outside of these tags will not be sent to the LLM, but may be used for providing extra context, notes, etc.

Body Features​

Within your body, you'll be able to:
